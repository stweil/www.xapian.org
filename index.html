The Xapian Project


Welcome to the pages of the Xapian project.

Xapian is not an application, but a library that enables developers to use advanced information retrieval techniques in their own projects. This makes it an attractive alternative to "packaged" search engines that need to be extensively hacked in order to use it as a specialty search engine.

Features

Features of Xapian include:
  • Flexible indexing of documents;
  • Full access to the index: from term to document, but also from document to terms;
  • Good query possibilities, including Boolean and phrase search;
  • Advanced ranking techniques;
  • Relevance feedback.
The Xapian project continues development of the Omsee search engine library, as published under the GPL by Brightstation plc. Since Brightstation will not continue development of Omsee under the GPL, the Xapian project unites developers and users of the library to pursue further development of the state-of-the-art information retrieval principles underlying the Omsee design.

A previous incarnation of this project was called Omseek, but we have been informed by Brightstation's legal department that this was too similar to Omsee. After much deliberation, the project name was changed to Xapian.

Downloads

The latest release version is 0.5.0. There are 3 tarballs:
  • xapian-core: the Xapian library itself
  • xapian-examples: small example programs
  • omega: Omega - an application built on Xapian, consisting of indexers and a CGI search frontend.

Access to Non-Release Versions via CVS

The Xapian CVS tree (including file history from the original Omsee project) can be browsed online on the Xapian CVS server. To get the latest version of Xapian directly from our CVS, follow these steps:
  1. Log in anonymously into the CVS: cvs -d:pserver:anonymous@cvs.xapian.org:/cvsroot/xapian login (Just hit return when you are asked for the password.)
  2. Check out the latest version: cvs -z3 -d:pserver:anonymous@cvs.xapian.org:/cvsroot/xapian co xapian
  3. In the newly created xapian directory, issue the command ./buildall - this will run various developer tools to produce a source tree like you'd get from unpacking a release source tarball.
If you encounter any problems, contact the developers on the xapian-devel mailing list, or report bugs via the bug tracking system.
