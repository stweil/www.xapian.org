The Xapian Project


Welcome to the pages of the Xapian project.
Xapian is not an application, but a library that enables developers to use advanced information retrieval techniques in their own projects. This makes it an attractive alternative to "packaged" search engines that need to be extensively hacked in order to use it as a specialty search engine.

Features of Xapian include:

  • Flexible indexing of documents;
  • Full access to the index: from term to document, but also from document to terms;
  • Good query possibilities, including Boolean and phrase search;
  • Advanced ranking techniques;
  • Relevance feedback.
The Xapian project will continue development of the Omsee search engine library, as published under the GPL by Brightstation plc. Since Brightstation will not continue development of Omsee under the GPL, the Xapian project unites developers and users of the library to pursue further development of the state-of-the-art information retrieval principles underlying the Omsee design.
A previous incarnation of this project was called Omseek, but we have been informed by Brightstation's legal department that this was too similar to Omsee. After much deliberation, the project name was changed to Xapian.

There are four mailing lists available for discussions:
xapian-discuss, a list for general discussions about the Xapian project. Subscribe yourself here.
xapian-devel, a list for technical discussions about the development of Xapian. Subscribe yourself here.
xapian-tracker, a lists that sends Tracker updates. Subscribe yourself here.
xapian-commits, a lists that sends CVS commits. Subscribe yourself here.

The original CVS tree of the Omsee project is preserved on the Xapian CVS server, though the Xapian developers have since made many modifications. To get the lastest version of Xapian directly from our CVS, follow these steps:

  1. Log in anonymously into the CVS: cvs -d:pserver:anonymous@cvs.xapian.org:/cvsroot/xapian login (Just hit return when you are asked for the password.)
  2. Check out the latest version: cvs -z3 -d:pserver:anonymous@cvs.xapian.org:/cvsroot/xapian co om
  3. In the newly created om directory, issue the command buildall.
If you encounter any problems, contact the developers on the xapian-devel mailing list, or report bugs via the bug tracking system.
